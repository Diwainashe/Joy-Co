 How Authentication Works

 1. Login (admin/index.html): User submits email + password → calls supabase.auth.signInWithPassword() → on success, redirects to /admin/dashboard.html
 2. Session persistence: Client initialized with persistSession: true, autoRefreshToken: true → session stored in localStorage, auto-refreshed. Login page calls getSession() on load and skips the
 form if already logged in.
 3. Route guard (admin/js/auth.js): Every admin page loads this on DOMContentLoaded. If no session → redirect to login. If session → set window.adminUser and inject a Logout button. Logout calls
 supabase.auth.signOut().
 4. Important: Protection is client-side only. Anyone who manipulates localStorage can bypass it. Acceptable for low-risk admin panel; not suitable for sensitive data.
